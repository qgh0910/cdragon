# 用户登录与知识库权限接口清单

更新日期：2026-07-02

本清单依据当前 `src/shuyixiao_agent/web_app.py` 静态路由重建，用于核对认证边界、知识库权限和 legacy 接口处置。新增或删除 FastAPI 路由时，应同步更新本清单。

## 分类口径

| 分类 | 含义 | 处理原则 |
| --- | --- | --- |
| 公开 | 匿名可访问接口 | 仅限页面、健康检查和登录入口 |
| 登录用户 | 需要有效登录态的业务接口 | 用户身份从 Session 派生 |
| 管理员 | 涉及全局资产、服务器路径或公共资源写操作 | 使用管理员校验及允许目录限制 |
| legacy 兼容 | 为旧调用方保留的接口 | 必须沿用当前授权解析，不信任客户端租户字段 |
| 下线策略 | 无法安全兼容的旧能力 | 前端迁移完成后返回 404/410 |

## 匿名白名单

| 路径 | 分类 | 说明 |
| --- | --- | --- |
| `/` | 公开 | 国内版页面 |
| `/global` | 公开 | 国际版页面 |
| `/api/health` | 公开 | 健康检查，不返回敏感配置 |
| `/api/auth/login` | 公开 | 登录入口 |

除上述入口外，`/api/*` 默认要求登录；管理员能力在登录基础上继续检查角色和资源范围。

## 服务器路径导入接口

| 方法 | 路径 | 风险字段 | 安全边界 |
| --- | --- | --- | --- |
| POST | `/api/rag/upload/file` | `file_path` | 管理员限定，并限制允许目录 |
| POST | `/api/rag/upload/directory` | `directory_path` | 管理员限定，并限制允许目录 |
| POST | `/api/lpos/contracts/parse` | `file_path` | 普通用户仅允许 `file_id`；管理员路径受允许目录限制 |

## 维护要求

- 新增路由时同步记录方法、路径、处理函数及权限分类。
- 服务器路径、删除、批量操作和公共知识库写入必须额外审查。
- legacy 接口不得因兼容性跳过当前认证和资源授权。
- 本清单只描述权限边界，不替代代码级鉴权测试。
